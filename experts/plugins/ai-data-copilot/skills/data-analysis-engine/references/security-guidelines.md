# 安全规范与数据保护指南

## 概述

本文档定义了智数分析引擎在数据处理全流程中的安全规范和数据保护策略。

---

## 执行安全

### 沙盒执行规则

#### 允许的操作
- ✅ 读取用户授权的数据源
- ✅ 执行 SELECT 查询
- ✅ 运行数据分析 Python 代码
- ✅ 生成文件（图表、报告）到指定输出目录
- ✅ 使用标准数据科学库

#### 禁止的操作
- ❌ INSERT / UPDATE / DELETE / DROP 等写操作
- ❌ 执行系统命令（os.system, subprocess）
- ❌ 访问网络（外部API调用）
- ❌ 读写沙盒外的文件系统
- ❌ 安装或卸载软件包
- ❌ 修改环境变量或系统配置

### SQL安全审查

## 数据保护

### 敏感数据识别

| 数据类型 | 识别规则 | 处理方式 |
|---------|---------|---------|
| 身份证号 | 15/18位数字，校验位 | 脱敏：保留前3后4 |
| 手机号 | 11位，1开头 | 脱敏：保留前3后4 |
| 邮箱 | 含@的字符串 | 脱敏：用户名部分隐藏 |
| 银行卡号 | 16-19位数字 | 脱敏：保留前6后4 |
| 姓名 | 人名字段 | 根据场景决定是否脱敏 |
| 地址 | 详细地址 | 模糊化到城市级别 |
| 密码/密钥 | password/secret/token字段 | 完全隐藏，不出现在输出 |

### 脱敏处理

```python
# 自动脱敏策略
def mask_sensitive(value, data_type):
    strategies = {
        "id_card": lambda v: v[:3] + "***" + v[-4:],
        "phone": lambda v: v[:3] + "****" + v[-4:],
        "email": lambda v: v[0] + "***@" + v.split("@")[1],
        "bank_card": lambda v: v[:6] + "****" + v[-4:],
        "name": lambda v: v[0] + "*" * (len(v)-1),
    }
    return strategies.get(data_type, lambda v: "***")(value)
```

## 输出安全

### 报告中的数据保护

1. **聚合优先**：优先展示聚合统计结果，避免暴露明细记录
2. **采样脱敏**：必须展示明细时，对敏感字段自动脱敏
3. **行数限制**：明细数据展示不超过100行
4. **审计日志**：记录每次数据访问的时间、范围和用途

### 代码输出保护

```python
# 输出清洗
output_protection = {
    "移除连接串": "替换密码部分为 ***",

    "移除Token": "所有token/api_key替换为 <redacted>",
}
```

## 隐私保护原则

### 数据处理合规

| 原则 | 实施 |
|------|------|
| 目的限制 | 仅用于用户指定的分析目的 |
| 数据最小化 | 仅获取完成分析所需的最少数据 |
| 存储限制 | 分析完成后不保留原始数据副本 |
| 完整性保护 | 不修改源数据 |
| 透明性 | 告知用户访问了哪些数据 |

### 跨境数据提醒

当检测到数据可能涉及以下场景时，主动提醒用户：
- 数据包含海外用户的个人信息
- 数据源服务器位于境外
- 分析结果可能需要跨境传输

## 异常处理安全

### 错误信息脱敏

```python
# 错误信息处理规则
error_handling = {
    "连接错误": "隐藏具体主机名和端口，仅报告'数据库连接失败'",
    "认证错误": "不暴露用户名，仅报告'认证失败，请检查凭证'",
    "查询错误": "保留SQL错误信息，但隐藏表名中的敏感部分",
    "系统错误": "记录完整错误到内部日志，向用户展示友好消息",
}
```

### 安全事件响应

当检测到以下情况时立即终止操作并报告：
1. SQL注入尝试
2. 权限越界访问
3. 异常大量数据导出
4. 恶意代码注入
5. 资源耗尽攻击

## 审计与可追溯

### 操作日志

每次分析操作记录以下信息（不含具体数据）：
- 操作时间
- 操作类型（查询/分析/生成）
- 数据源标识（不含凭证）
- 数据范围（行数/时间范围）
- 执行状态（成功/失败）
- 输出类型（报告/图表/数据）

### 可复现性

- 所有分析代码和SQL可追溯
- 随机种子记录确保结果可重复
- 分析参数和配置完整保留
- 数据版本标记（时间戳或快照ID）
