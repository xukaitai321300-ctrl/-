# 数据分析模式与最佳实践

## 概述

本文档定义了智数分析引擎支持的标准分析模式、工作流模板和最佳实践指南。

---

## 分析模式索引

### 模式1：Text2SQL 自然语言查询

**适用场景**：用户用自然语言提问，需要从数据库中获取答案

**工作流**：
```
用户问题 → 意图识别 → Schema匹配 → SQL生成 → 安全审查 → 执行 → 结果解读
```

**关键步骤**：

1. **意图识别**
   - 提取实体：时间范围、目标指标、过滤条件、排序方式
   - 识别查询类型：统计聚合 / 明细查询 / 趋势分析 / 排名对比

2. **Schema匹配**
   - 将用户提到的概念映射到数据库字段
   - 处理同义词和别名（如"收入"→"revenue"字段）
   - 识别需要JOIN的表

3. **SQL生成策略**
   - 简单查询：直接生成
   - 复杂查询：使用CTE分步构建
   - 不确定时：生成2-3个候选SQL并说明差异

4. **安全审查**
   - 确认是SELECT语句
   - 检查是否有LIMIT保护
   - 验证不涉及敏感字段

**示例**：
```
用户："上个月销售额最高的前10个产品是什么？"

意图：聚合查询 + 排序 + TopN
实体：时间=上个月, 指标=销售额, 排序=降序, N=10
表：products JOIN orders

SQL:
SELECT p.product_name, SUM(o.amount) as total_sales
FROM orders o
JOIN products p ON o.product_id = p.id
WHERE o.order_date >= DATE_SUB(CURDATE(), INTERVAL 1 MONTH)
  AND o.order_date < CURDATE()
GROUP BY p.product_name
ORDER BY total_sales DESC
LIMIT 10;
```

### 模式2：数据画像（Data Profiling）

**适用场景**：首次接触数据集，需要全面了解数据特征

**工作流**：
```
数据加载 → 元信息统计 → 字段分析 → 质量评估 → 相关性 → 画像报告
```

**分析维度**：

| 维度 | 统计项 |
|------|--------|
| 基本信息 | 行数、列数、大小、时间跨度 |
| 数据类型 | 数值型、分类型、日期型、文本型占比 |
| 缺失值 | 每列缺失率、缺失模式（随机/系统性） |
| 唯一值 | 基数（cardinality）、重复率 |
| 数值分布 | 均值、中位数、标准差、偏度、峰度 |
| 分类分布 | Top-N类别、长尾比例 |
| 异常值 | IQR方法或Z-score检测 |
| 相关性 | 数值列间的Pearson/Spearman相关矩阵 |

**输出模板**：
```python
profile = {
    "overview": {
        "rows": N,
        "columns": M,
        "memory_usage": "X MB",
        "duplicated_rows": K,
        "missing_cells_pct": "X%"
    },
    "columns": [
        {
            "name": "col_name",
            "dtype": "int64",
            "missing_pct": 0.05,
            "unique_count": 150,
            "stats": {"mean": 42.3, "std": 12.1, "min": 1, "max": 100}
        }
    ],
    "correlations": [[1.0, 0.8], [0.8, 1.0]],
    "quality_score": 0.87  # 0-1综合质量评分
}
```

### 模式3：趋势分析

**适用场景**：分析指标随时间的变化趋势

**工作流**：
```
时间序列提取 → 趋势分解 → 周期识别 → 异常检测 → 预测（可选） → 可视化
```

**分析方法**：

1. **趋势分解**（STL分解）
   - 趋势成分（Trend）
   - 季节成分（Seasonal）
   - 残差成分（Residual）

2. **变点检测**
   - 识别趋势发生显著变化的时间点
   - CUSUM算法或Bayesian方法

3. **同比/环比计算**
   ```python
   yoy = (current - same_period_last_year) / same_period_last_year * 100
   mom = (current - previous_period) / previous_period * 100
   ```

4. **预测方法选择**
   | 数据特征 | 推荐方法 |
   |---------|---------|
   | 线性趋势，无季节性 | 线性回归 |
   | 有季节性 | SARIMA / Prophet |
   | 多变量影响 | VAR / XGBoost |
   | 数据量少 | 移动平均 / 指数平滑 |

---

### 模式4：对比分析

**适用场景**：比较不同组/时期/方案的差异

**工作流**：
```
定义对比维度 → 数据分组 → 指标计算 → 差异检验 → 效应量化 → 可视化
```

**方法选择**：

| 对比类型 | 统计方法 |
|---------|---------|
| 两组均值对比 | t检验 |
| 多组均值对比 | 方差分析（ANOVA） |
| 比例对比 | 卡方检验 / Fisher精确检验 |
| 分布对比 | KS检验 / Mann-Whitney U |
| 前后对比 | 配对t检验 / Wilcoxon |

---

### 模式5：知识增强分析

**适用场景**：结合文档知识库和结构化数据进行深度分析

**工作流**：
```
数据分析结论 + 知识库检索 → 交叉验证 → 上下文丰富 → 深度解读
```

**增强策略**：
- 数据结论 + 行业报告佐证
- 异常值 + 知识库查找可能原因
- 趋势变化 + 检索相关事件
- 指标含义 + 知识库中的定义和基准

## 最佳实践

### 数据质量处理

```python
# 标准数据质量处理流程

# 1. 缺失值处理策略
missing_strategies = {
    "少量随机缺失": "删除行或均值/中位数填充",
    "大量系统缺失": "标记为特殊类别或建模预测",
    "关键字段缺失": "向用户报告，建议数据补充",
}

# 2. 异常值处理
outlier_methods = {
    "IQR方法": "Q1 - 1.5*IQR ~ Q3 + 1.5*IQR",
    "Z-score": "|z| > 3 视为异常",
    "领域知识": "基于业务逻辑判断合理范围",
}

# 3. 数据一致性
consistency_checks = [
    "日期格式统一",
    "分类值标准化（大小写、空格）",
    "数值单位统一（元/万元/亿元）",
    "编码一致性（UTF-8）",
]
```

### 分析结论规范

1. **量化表达**：所有结论附带具体数值
   - ✅ "Q4营收同比增长23.5%，达到1,850万元"
   - ❌ "Q4营收有明显增长"

2. **因果审慎**：区分相关和因果
   - ✅ "广告投入与销售额呈强正相关（r=0.85），但无法确认因果关系"
   - ❌ "广告投入导致销售额增长"

3. **置信标注**：统计结论标注显著性
   - ✅ "A组转化率（4.2%）显著高于B组（3.1%），p<0.01"
   - ❌ "A组比B组好"

4. **局限声明**：主动说明分析局限
   - 数据时间范围的限制
   - 样本量是否足够
   - 可能的选择偏差

### 性能优化

| 场景 | 优化策略 |
|------|---------|
| 大表全扫描 | 添加WHERE条件或采样 |
| 多表JOIN | 确认有索引，考虑物化视图 |
| 复杂聚合 | CTE分步计算，避免嵌套子查询 |
| 重复计算 | 缓存中间结果 |
| 大结果集 | 数据库端聚合，仅拉取摘要 |

## 错误处理与恢复

### 常见错误及处理

| 错误类型 | 处理方式 |
|---------|---------|
| 连接超时 | 重试3次，间隔递增（1s, 3s, 5s） |
| 查询超时 | 简化查询或添加更强的过滤条件 |
| 权限不足 | 报告需要的权限，建议联系DBA |
| 内存溢出 | 分批处理或降低数据量 |
| 语法错误 | 检查数据库方言，自动修正 |
| 数据为空 | 给出友好提示，建议检查过滤条件 |

### 降级策略

当首选方案不可用时的降级处理：

```
全量分析 → 采样分析 → 聚合统计 → 提示用户数据过大
精确计算 → 近似计算 → 估算 → 告知精度限制
实时查询 → 缓存数据 → 历史快照 → 告知数据时效
```
