# 蒸馏流程 — 日记忆 → 长期记忆

## 什么是蒸馏

蒸馏 = 从日记忆（原始日志）中提取值得长期保留的内容，写入 MEMORY.md。

日记忆是流水账，MEMORY.md 是精华。就像人类翻日记本，把重要的感悟写进"人生经验"。

## 何时蒸馏

- **定期**：每 2-3 天一次（可在心跳中执行）
- **触发式**：发生重大事件时立即更新

## 蒸馏步骤

1. **扫描**：读取最近 3-7 天的 `memory/YYYY-MM-DD.md`
2. **筛选**：识别以下类型的内容
   - 重大决策及理由
   - 关键教训（编号，永不回收）
   - 重要认知/发现
   - 策略变化
   - 系统架构变更
3. **提炼**：用简洁语言写入 MEMORY.md 对应章节
4. **清理**：MEMORY.md 中过时信息降级或归档
5. **记录**：在 MEMORY.md 头部更新 `上次提炼` 日期

## MEMORY.md 结构模板

见 `assets/MEMORY.md`。

## 归档流程

1. 找到 MEMORY.md 中 >30 天未被引用的章节
2. 移到 `memory/archive/YYYY-MM-DD-chapter-name.md`
3. 在 MEMORY.md 原位留一行占位：`### ~~章节名~~ → 已归档 (YYYY-MM-DD)`

## 自动化（可选）

可配置 cron 定时执行蒸馏。推荐频率：每天 1 次，凌晨低峰时段。

蒸馏脚本示例逻辑：
```bash
# 伪代码 — 具体实现因 Agent 而异
1. 读取最近 7 天的 memory/*.md
2. 提取关键词和重大事件
3. 对比 MEMORY.md，更新/新增/归档
4. 写入 MEMORY.md
5. 记录 distill_log.txt
```

## 注意事项

- 蒸馏是**有损压缩** — 保留决策和教训，丢弃过程细节
- 教训编号**永不回收** — 即使教训已过时，编号留给未来追溯
- 动态信息（持仓、策略参数）超过 14 天未更新应降级
- 蒸馏后不要删除日记忆文件 — 它们是原始记录
