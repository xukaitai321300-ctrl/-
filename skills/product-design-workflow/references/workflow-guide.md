# 产品设计工作流详细指南

## 完整工作流程

### 第一步：生成头脑风暴点子

**输入**：无（或指定主题方向）
**输出**：3个idea目录，每个包含`idea.md`

**要求**：
- 第1个：市场行情方向
- 第2个：A股方向
- 第3个：基金方向
- 1个小白向，2个普通向

**idea.md 内容要点**：
1. 功能初衷 - 解决什么痛点
2. 一句话卖点 - 核心价值
3. 用户痛点分析 - 详细分析
4. 功能描述 - 使用场景、操作流程、输出结果
5. Z世代表达设计 - 网感设计
6. 竞品参考 - 差异化优势

### 第二步：生成PRD文档

**输入**：idea.md
**输出**：product-design.md

**PRD文档内容要点**：
1. 需求概述 - 背景、目标、目标用户
2. 核心功能列表 - 优先级表格
3. 页面结构说明 - 页面清单、原型说明
4. 交互逻辑说明 - 操作流程、异常处理、跳转
5. 文案规范 - 语气、示例、禁用词
6. 视觉风格要求 - 色调、风格、适配

### 第三步：制作HTML原型

**输入**：product-design.md
**输出**：demo/目录，包含HTML页面

**HTML模板选择**：
- `base.html` - 简单单页
- `list-template.html` - 列表页
- `detail-template.html` - 详情页
- `tool-template.html` - 工具页

**HTML页面要求**：
1. 必须引入公共Header和Footer
2. 必须引入`assets/css/common.css`
3. 必须引入`assets/js/header.js`和`assets/js/footer.js`
4. 右下角必须包含Idea介绍浮层
5. 所有资源使用相对路径

## 质量检查清单

### Idea检查
- [ ] 主题方向正确（市场行情/A股/基金）
- [ ] 受众配比正确（1小白向+2普通向）
- [ ] 标签格式正确
- [ ] 内容完整，涵盖所有要点
- [ ] 从用户需求出发，不纠结技术实现

### PRD检查
- [ ] 需求概述清晰
- [ ] 功能列表有优先级
- [ ] 页面结构明确
- [ ] 交互逻辑完整
- [ ] 文案规范明确
- [ ] 视觉风格有要求

### HTML检查
- [ ] 正确引入公共资源
- [ ] Header和Footer显示正常
- [ ] Idea介绍浮层存在
- [ ] 所有链接可跳转
- [ ] 样式统一，符合设计规范

## 常见问题

### Q: 如何选择HTML模板？
A: 根据PRD中的页面类型选择：
- 单页展示 → base.html
- 列表/卡片 → list-template.html
- 文章/详情 → detail-template.html
- 工具/计算器 → tool-template.html

### Q: 小白向内容有什么要求？
A: 绝对通俗易懂，禁止专业术语，多用比喻，网感强。

### Q: 普通向内容有什么要求？
A: 正常表达即可，无需特别小白，但也要避免过于专业的术语堆砌。

### Q: 可以修改公共样式吗？
A: 不可以。Header、Footer、common.css为全局固定，如需调整统一修改公共文件。
