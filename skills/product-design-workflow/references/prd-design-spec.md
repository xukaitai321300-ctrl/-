# 产品PRD设计规范
## 🔹 整体要求
- PRD必须完全基于对应idea的内容生成，不能偏离点子的核心需求
- 内容必须可直接交付给研发使用，不需要额外沟通
- 优先级明确，核心功能和次要功能区分清楚
- 移动端优先，所有设计默认支持响应式

---
## 🔹 内容模块必填规范
### 1. 头部元数据
```
# 产品设计文档：[点子中文名称]
**版本**: v1.0  
**日期**: YYYY-MM-DD  
**需求来源**: 头脑风暴点子
```

### 2. 需求概述（必填）
#### 2.1 需求背景
- 对应idea的痛点部分，简要说明为什么要做这个功能
#### 2.2 需求目标
- 明确上线后要达到的效果，可量化
#### 2.3 目标用户
- 和idea的目标用户保持一致

### 3. 核心功能列表（必填）
- 使用表格形式，优先级分为P0（必须有）、P1（应该有）、P2（可以有）
- 每个功能点必须说明清楚是什么，边界明确
- P0功能不能超过3个

### 4. 页面结构说明（必填）
#### 4.1 页面清单
- Demo页面控制在1~3个，不能超过3个
- 每个页面必须明确路径和用途
#### 4.2 核心页面原型说明
- 每个页面分区域说明内容、布局
- 不需要画原型图，用文字描述清楚每个区域展示什么内容即可

### 5. 交互逻辑说明（必填）
- 核心操作流程必须一步一步写清楚
- 异常情况必须说明处理方式（比如网络错误、数据为空）
- 页面之间的跳转逻辑必须明确

### 6. 文案规范（必填）
- 整体语气统一：要么小白友好，要么专业严谨，不能混合
- 核心文案示例必须给出
- 明确禁止使用的词汇（比如引导投资类词汇：推荐、买入、布局等）

### 7. 视觉风格要求（必填）
#### 7.1 品牌色规范（强制统一）
| 颜色类型 | 色值 | 应用场景 |
|---------|------|----------|
| 全局主色调（品牌紫） | `#695ddf` | 所有主按钮、标签、重点强调元素、品牌相关视觉元素、选中态、链接色 |
| 主色调浅版 | `#8b82eb` | hover态、淡色背景、轻量强调元素 |
| 主色调深版 | `#584bc0` | active态、深色场景主色、点击态 |
| 主色调透明版 | `#695ddf1a` | 淡色背景、阴影、选中态背景 |
| 辅助色（暖橙） | `#FF7A45` | 渐变搭配、醒目提示、热门标签、金额数字 |
| 辅助色（青绿） | `#5cdbd3` | 渐变搭配、成功态、清新风格场景 |
| 中性色 | 白/浅灰/深灰 | 正文、背景、边框等常规元素 |

#### 7.2 官方常用渐变组合（直接复用）
| 渐变类型 | 渐变代码 | 应用场景 |
|---------|----------|----------|
| 主按钮渐变 | `linear-gradient(135deg, #695ddf, #584bc0)` | 所有主操作按钮 |
| 标题渐变 | `linear-gradient(135deg, #695ddf, #ff7a45, #5cdbd3)` | 大标题、宣传类文案 |
| 暖系渐变 | `linear-gradient(135deg, #695ddf, #ff7a45)` | 卡片、促销类场景 |
| 冷系渐变 | `linear-gradient(135deg, #695ddf, #5cdbd3)` | 清新风格、数据类场景 |

#### 7.3 禁止规则
- 禁止使用老气的深蓝色、暗红色等传统金融产品常用的沉重色调
- 禁止随意修改品牌色值，所有颜色必须从上述规范中选择
#### 7.2 整体风格要求
- 年轻化：简洁、现代、活泼，符合Z世代审美，拒绝传统金融产品的严肃、老气风格
- 轻量化：留白充足，不要过度设计，信息层级清晰
- 友好感：可以适当加入趣味化的图标、emoji元素，降低用户距离感
#### 7.3 适配要求
- 默认移动端优先，所有设计首先保证手机端体验优秀
- 支持响应式适配PC端，PC端布局合理
- 字号最小不小于12px，保证移动端可读性
#### 7.4 元素规范
- 图标：使用线性简约风格图标，禁止使用拟物化、厚重的图标
- 字体：默认使用无衬线字体，标题可以适当加粗，正文清晰易读
- 圆角：卡片、按钮统一使用圆角设计，增强友好感
- 阴影：适当使用轻阴影，增加层次感，不要使用过重的阴影
