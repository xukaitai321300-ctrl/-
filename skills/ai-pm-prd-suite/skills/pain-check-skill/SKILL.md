# 内部 Skill：痛点评估

> 此 Skill 仅供 Sub Agent 内部调用，不直接暴露给用户

---

## 功能

评估一个痛点是否适合用 AI 解决。

---

## 输入

```json
{
  "pain_point": "痛点描述",
  "current_solution": "当前解决方案",
  "user_expectation": "用户期望"
}
```

---

## 输出

```json
{
  "score": 22,
  "max_score": 25,
  "dimensions": [
    {"name": "问题标准化", "score": 5, "note": "大部分是 FAQ"},
    {"name": "数据质量", "score": 4, "note": "数据存在但参差"},
    {"name": "错误容忍", "score": 4, "note": "用户能接受偶尔出错"},
    {"name": "实时性", "score": 5, "note": "需要 24/7"},
    {"name": "成本敏感", "score": 4, "note": "人工成本高"}
  ],
  "conclusion": "适合",
  "recommendation": "推荐使用 AI 方案，建议混合模式"
}
```

---

## 评估维度

| 维度 | 评分标准 |
|------|----------|
| 问题标准化 | 5=完全标准化，1=完全不标准化 |
| 数据质量 | 5=数据充足且高质量，1=无数据 |
| 错误容忍 | 5=完全容忍，1=零容忍 |
| 实时性 | 5=必须 24/7，1=无实时要求 |
| 成本敏感 | 5=成本压力很大，1=成本不敏感 |

---

## 结论判断

- 总分 ≥ 20：适合用 AI
- 总分 15-19：有条件适合，需讨论风险
- 总分 < 15：不适合，建议其他方案
