# 内部 Skill：置信度设计

> 此 Skill 仅供 Sub Agent 内部调用，不直接暴露给用户

---

## 功能

根据用户输入，设计置信度机制。

---

## 输入

```json
{
  "product": "AI 客服助手",
  "confidence_threshold": 0.7,
  "human_transfer_rate": 0.2,
  "triggers": ["置信度低", "关键词命中", "用户反馈"]
}
```

---

## 输出

```markdown
# 置信度机制设计

## 置信度分级
| 等级 | 范围 | 含义 | 呈现方式 |
|------|------|------|----------|
| 高 | 0.8-1.0 | 确定正确 | 直接展示 |
| 中 | 0.5-0.8 | 可能正确 | 展示+提示 |
| 低 | 0.3-0.5 | 不确定 | 候选+提示人工 |
| 极低 | <0.3 | 无法判断 | 明确告知 |

## 人工介入触发器
| 触发条件 | 阈值 | 动作 |
|----------|------|------|
| 置信度 | <0.7 | 转人工 |
| 关键词 | 投诉/退款 | 转人工 |
| 用户反馈 | 连续2次不对 | 转人工 |

## PM 决策确认
- 置信度阈值：0.7
- 转人工比例上限：20%
```
